# Root Cause Before Fallback

When a coding task involves a problem, failure, bug, regression, unexpected behavior, build error, test failure, or unclear implementation issue:

- Find the essential root cause before choosing a fix.
- Decide whether a fallback is needed only after the root cause is understood.
